# Agricultural Subsistence Replication Package

This package reproduces the tables and figures used in the paper from the original files in `raw_data/`.

## Folder structure

```text
raw_data/      Original downloaded source files. Do not edit these files.
clean_data/    Files generated by `code/00_clean_data.py`. Analysis scripts read only this folder.
code/          Python scripts for cleaning, calibration, regressions, maps, and checks.
derived/       Intermediate files created by the analysis scripts.
outputs/       Final manuscript-facing tables and figures.
```

## Raw data included

The `raw_data/` folder is included in this package exactly so the workflow can begin from the original source files.
It contains:

- `raw_data/ipums/`: IPUMS USA extract data, DDI/XML metadata, codebook, and command files.
- `raw_data/usda/`: USDA ERS national productivity table and state productivity accounts.
- `raw_data/income/`: USDA ERS farm household income file.
- `raw_data/quickstats/`: USDA NASS Quick Stats crop acreage and fieldwork-days-suitable downloads.
- `raw_data/snap/`: USDA FNS SNAP participation file.
- `raw_data/population/`: Census state population estimates.
- `raw_data/data_directory.docx` and `.pdf`: human-readable documentation for the raw files.

## Method choices implemented in code

- Calibration uses `Aa_labor_productivity_index`, defined as USDA ERS total agricultural output quantity index divided by labor input quantity index.
- The old within-state 1960-normalized productivity series is not used for calibration.
- Baseline calibration recovers state subsistence indices from the 1960 and 1970 early moments.
- Robustness calibration checks also use 1960 only and 1960/1970/1980.
- IV and interaction regressions read the baseline raw-productivity calibration output.
- SNAP outcome is the log SNAP participation rate: SNAP persons divided by the 2025 Census population estimate.
- Historical crop instruments use `log(1 + average planted acreage)` over the historical window used in the scripts.
- Suitability is based on USDA NASS Quick Stats fieldwork days suitable.

## How to run

Install dependencies:

```bash
pip install -r requirements.txt
```

Then run:

```bash
python run_all.py
```

The first script cleans the original source files from `raw_data/` and writes `clean_data/`. All later scripts use only `clean_data/` and the derived calibration outputs.

## Outputs generated

Final outputs are written to:

```text
outputs/motivation/
outputs/calibration/
outputs/regression/
outputs/maps/
```

## Important note on runtime

The IPUMS raw data file is large. The cleaning script reads it in chunks, so it may take several minutes depending on the computer.


### Calibration figures

The calibration script always produces the aggregate data-vs-model figure and a required folder of state-level model-fit figures:

`calibration_output/state_model_fit_all/`

Each PNG in that folder compares the observed agricultural employment share with the calibrated model-implied share for one state. These figures are generated from the baseline raw-productivity calibration using 1960 and 1970 as calibration moments.

# County-Level Historical Soybean Replication

This package reproduces the county-level historical soybean reduced-form exercise.

## Data

Raw NHGIS data are already included in:

```text
raw_data/nhgis/
```

The key NHGIS variable is:

```text
AB4X001 = Soybean acreage, 1919
```

The main treatment variable is:

```text
log(1 + soybean acres per 1,000 land acres)
```

where:

```text
soybean acres per 1,000 land acres = 1000 * soybean_acres_1919 / land_acres
```

To find the source manually in NHGIS:

1. Go to `https://data2.nhgis.org/`.
2. Use **Source Tables**.
3. Set `Geographic Levels = County`.
4. Set `Years = 1920`.
5. Set `Topics includes Crops and Other Farm Products`.
6. Select `NT045. Grains and Seeds -- Acreage by Type`, `1920_cAg`.

The package downloads ACS 2024 county SNAP and control variables from the Census API.

It also downloads the Census 2024 National Counties Gazetteer file to add county land area:

```text
https://www2.census.gov/geo/docs/maps-data/data/gazetteer/2024_Gazetteer/2024_Gaz_counties_national.zip
```

The raw Gazetteer zip is stored at:

```text
raw_data/census_gazetteer/census_2024_county_gazetteer.zip
```

## Requirements

Install Python 3.10 or newer. Then install:

```powershell
pip install -r requirements.txt
```

## Run

Set your Census API key:

```powershell
$env:CENSUS_API_KEY="your_census_key"
```

Run:

```powershell
python code\county_level.py
```

You can also open the same file in Spyder:

```text
code/county_level.py
```

Run the file. If `CENSUS_API_KEY` is not already set, the script will ask you to enter the key.

Note: the current local testing copy of `code/county_level.py` contains a temporary `os.environ["CENSUS_API_KEY"] = ...` line. Remove that line before publishing or sharing the replication package.

## Outputs

The script writes outputs to:

```text
clean_data/acs_county_2024.csv
clean_data/county_land_area_2024.csv
clean_data/nhgis_soybean_1919.csv
clean_data/county_historical_soybean_panel.csv
output/tables/historical_soybean_snap_results.csv
output/tables/historical_soybean_snap_results.tex
output/tables/historical_soybean_socioeconomic_outcomes.csv
output/tables/historical_soybean_socioeconomic_outcomes.tex
```

All regressions include state fixed effects and state-clustered standard errors. The SNAP table reports all combinations of four control groups: land area, population, demographics, and socioeconomic controls.
